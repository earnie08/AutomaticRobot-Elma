5 Project Goal :
6 ---
7 A lifetime security guard robot wandering around on a 2D array map and helping you detect the intruder until you shut it down. The user can start or stop the robot by using keyboards. Also, the user can adjust most of the robot set in [jason file](robot_setting.json).
8 
9 There might have an intruder showing on the map in a random location(default setting is changing location for every 5 seconds). Besides that, the robot will stop and make noise after it found an intruder in its detecting area. If the intruder is too close to the robot, the robot will use 2x speed escaping on the map.
10 
11 The robot will continue wandering around, detecting the intruder and recharging itself and it won't be shut down before you press 'q' to quit.

91  ### <a id="robot"></a>__Robot__
92  Robot class is inherited to elma's state machine class. In the robot state machine, there are serveral states and functions.
93 
94  States:
95 
96  <img src="images/RobotStateMachine.png" width="100%"/>
97 
98  - Off ([off.h](off.h)): Press 's' to start the robot.
99  - Wander ([wander.cc](wander.cc)):
100  1. Detect the intruder's position. If the intruder is in the detection area, set the battery consumption rate to the half of the default rate.
101  2. Detect battery status.
102  3. The robot will move 1 step each time, the direction is up, left, right and down.
103  - Noise ([noise.cc](noise.cc)):
104  1. Detect the intruder's position. If the intruder is out of the detection area, reset the consumption rate, turn off the noise and back to wander state.
105  2. Detect whether the intruder is in the proximity area. If true, sett the battery consume rate to 2 times of default rate.
106  3. Detect battery status.
107  4. Robot stop wandering.
108  5. Making noise.
109  - Evade ([evade.cc](evade.cc)):
110  1. Detect the intruder's position. If the intruder is out of the proximity area, reset the consumption rate and back to noise state.
111  2. Detect battery status.
112  3. robot will move 2 steps each time to escape. The direction is random.
113  - Finding ([finding.cc](finding.cc)):
114  1. Detect robot is in charge station or not. If it's in the charge station, start charging.
115  2. Plan the path to the charge station. The routing logic is that the robot will go x-axis first until it has the same x value as the charge station. If the intruder showed up on the route and it's in front of the robot, the robot will detour or try to go y-axis first.

119  |:-----------:|:----------:|:----------------:|
120  |<img src="images/PathToChargeStation.png"/> | <img src="images/DetourToChargeStation.png"/> | <img src="images/Y-AxisFirstDetour.png"/> |
121 
122  - Charging ([charging.cc](charging.cc)):
123  1. Charging rate is 2 times of the default consume rate.
124  2. If the battery is full, set the robot position at the last location.
125 
126  Functions:
127  - __batteryDetection__ : Detect whether the battery can support robot go back to charge station. Reserved 2 steps for the robot to avoid it needs to detour.
128 
129  - __intruderDetection__ : Detect whether intruder's location is in the detection area, the default is a 4x4 area from the robot's location on the map.
130 
131  - __proximityDetection__ : Detect whether intruder's location is too close to the robot, the default is detecting 2x2 area around the robot's location on the map.

145  ### <a id="setting"></a>__Customize Setting__
146  Customize setting for the robot.
147 
148  There are some restriction for customize setting:
149  - Robot Name: should be __string__ type.
150  - Map Size: should be in range __5__ to __20__. (for the best visualization)
151  - Battery Size: should __greater than consume rate__ and __0__
152  - Battery Consume Rate: should __greater than 0__.
153  - Charge Station: x and y shold __smaller than map size__ and __greater or equal to 0__.
154  - Robot Position: x and y shold __smaller than map size__ and __greater or equal to 0__.
155  - Robot Detection Area: should be __smaller or equal to map size__.
156  - Robot Detection Area: should be __smaller or equal to map size__.
157 
158  **Battery size divided to two times consume rate should be smaller than the steps that the robot needs to go back to charge station.
159 
160  Example:
161  ~~~
162  {
163  "Robot Name": "Security Robot",
164  "Map Size": 20,
165  "Battery Size": 100,
166  "Battery Consume Rate": 2,
167  "Charge Station": [19,19],
168  "Robot Position": [0,0],
169  "Robot Detection Area": 20,
170  "Robot Proximity Area": 20
171  }
172  ~~~
